# Penalty Shootout — 产品目标与开发状态

本文件用于在任意新开发环境中恢复完整上下文。开始开发前请先阅读 AGENTS.md 和本文件，
不要把目标缩减为当前已经完成的功能。

## 当前技术与运行方式

- 仓库：https://github.com/oIdman/football-game
- 运行时：Godot 4.7 + GDScript
- 启动：用 Godot 4.7 打开 project.godot，运行 Main.tscn
- 分辨率：480×720（竖屏）
- UI：Control 节点 + CanvasLayer
- 版本管理：Git 标签

|------|------|------|

## 版本详细记录

### v0.1.0 — 最小 2D 点球游戏
- Godot 4.7 2D 点球游戏
- 球门视角：球门后方望向整个球门（俯视 2D 渲染）
- 守门员：键盘（方向键/WASD）和鼠标控制，在球门框内自由移动
- 足球：从屏幕正中心出现，大小由小到大，随机攻向球门左侧或右侧
- 轨迹拖尾：Line2D 显示球的运动轨迹
- 计分：守门员碰到球 +1 分，继续发球，球进门则结算
- 开始按钮、失败界面、重新开始
- 运行时生成纹理（无需外部图片资源）

## 下一版本分析

当前版本为 v0.1.0，已完成最小可用点球游戏。

### 可考虑的优化方向
- 增加音效（扑救、进门、背景）
- 增加难度递增速度设置
- 美化画面（真实球门贴图、草地纹理）
- 增加球门柱碰撞检测（打中门柱反弹）
- 增加连击奖励动画
- 移动端触屏适配

## 持续开发要求

- 每轮按照"最小可用功能 + 渐进增强"的方向推进。
- 新模块必须接入现有体系，不能只做静态演示。
- 每次修改必须用 Godot 编辑器或 CLI 打开项目验证无报错。
- 每轮开始时通过 GitHub Issues 检查；可执行的回复并纳入版本，需澄清的回复问题并保持开放。
- Issue 处理结果同时写入本文件和 README（双写）。
- 提交信息使用语义化风格（feat/fix/chore），创建对应版本标签并推送。

## 完整产品目标

一款最小 2D 点球游戏，模拟球门视角的守门员接球体验。

## 最终完成标准

1. ⬜ 核心玩法：守门员移动 + 球随机发射 + 计分 + 失败结算（v0.1.0 ✅）
2. ⬜ 音效系统：扑救音效、进门音效、背景音乐
3. ⬜ 视觉美化：球门贴图、草地、动画效果
4. ⬜ 难度系统：速度递增、不同射门模式
